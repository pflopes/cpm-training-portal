Demo Project

A complete .NET solution demonstrating Central Package Management in a fintech context.

FintechDemo Solution Structure

The demo project includes the following components:

  • FintechDemo.Core - Domain models and interfaces for transaction management
  • FintechDemo.Infrastructure - Data access layer with repository pattern
  • FintechDemo.Api - REST API endpoints for transaction operations
  • FintechDemo.Tests - Comprehensive unit tests using xUnit and Moq

Key Features

  • Centralized package versioning with Directory.Packages.props
  • Transitive pinning enabled for consistent dependency management
  • Global package references for code analyzers
  • Clean architecture with clear separation of concerns
  • Security packages for authentication and JWT bearer tokens
  • Entity Framework Core for data access
  • Comprehensive test coverage with mocking and assertions

Directory.Packages.props Example

<Project>
  <PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
    <CentralPackageTransitivePinningEnabled>true</CentralPackageTransitivePinningEnabled>
  </PropertyGroup>
  <ItemGroup>
    <!-- Common Packages -->
    <PackageVersion Include="Microsoft.Extensions.DependencyInjection" Version="8.0.0" />
    <PackageVersion Include="Newtonsoft.Json" Version="13.0.3" />
    <!-- Security Packages -->
    <PackageVersion Include="Microsoft.AspNetCore.Authentication.JwtBearer" Version="8.0.0" />
  </ItemGroup>
  
  <ItemGroup>
    <GlobalPackageReference Include="SonarAnalyzer.CSharp" Version="9.19.0.84025" />
  </ItemGroup>
</Project>

Project File Example

<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net8.0</TargetFramework>
  </PropertyGroup>
  
  <ItemGroup>
    <!-- No Version attribute needed -->
    <PackageReference Include="Microsoft.Extensions.DependencyInjection" />
    <PackageReference Include="Newtonsoft.Json" />
  </ItemGroup>
</Project>

Key Concepts

Central Version Definition

All package versions are defined in a single Directory.Packages.props file at the solution root, providing a single source of truth for dependency management.

Version Resolution

Projects reference packages without specifying versions. NuGet automatically resolves versions from the central file during restore and build operations.

Consistency Across Projects

All projects in the solution use identical package versions by default, ensuring consistent behavior and reducing compatibility issues.

Security Management

Security packages can be updated in one location, ensuring all projects immediately benefit from security patches and updates.

Flexibility with Overrides

When needed, specific projects can override versions using the VersionOverride attribute for special compatibility requirements.

Transitive Dependency Control

With transitive pinning enabled, you can control versions of indirect dependencies, ensuring complete dependency management.

Benefits for Fintech

Enhanced Security

Consistent security package versions across all projects reduce vulnerability risks and make security audits more straightforward.

Regulatory Compliance

Simplified audit trails and version tracking make it easier to meet strict fintech compliance requirements.

Operational Efficiency

Reduced time spent on dependency management and fewer merge conflicts related to package versions.

Risk Reduction

Lower chance of runtime errors due to version conflicts and better control over third-party code.
