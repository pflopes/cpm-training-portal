Migrating Existing Projects
Migration Steps
  1. Create Directory.Packages.props at solution root
  2. Enable central management with ManagePackageVersionsCentrally
  3. Extract package versions from all project files
  4. Remove version attributes from PackageReference elements
  5. Build solution and fix any version conflicts
  6. Test thoroughly before committing
Migration Tools

CentralisedPackageConverter - A CLI tool to automate migration:

# Install the tool globally
dotnet tool install CentralisedPackageConverter --global

# Convert your solution
central-pkg-converter /PATH_TO_SOLUTION
