Enabling Central Package Management
Step-by-Step Implementation
1
Create Directory.Packages.props at solution root
2
Set ManagePackageVersionsCentrally to true
3
Define package versions using PackageVersion elements
4
Update project files to use PackageReference without versions
5
Build the solution to apply centrally managed versions
# Create props file template (if available)
dotnet new packagesprops
Directory.Packages.props
<Project>
  <PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
  </PropertyGroup>
  <ItemGroup>
    <!-- Common Packages -->
    <PackageVersion Include="Newtonsoft.Json" Version="13.0.3" />
    <PackageVersion Include="Microsoft.Extensions.Logging" Version="8.0.0" />
  </ItemGroup>
</Project>
Project File (.csproj)
<Project Sdk="Microsoft.NET.Sdk">
  <ItemGroup>
    <!-- No Version attribute needed -->
    <PackageReference Include="Newtonsoft.Json" />
    <PackageReference Include="Microsoft.Extensions.Logging" />
  </ItemGroup>
</Project>
